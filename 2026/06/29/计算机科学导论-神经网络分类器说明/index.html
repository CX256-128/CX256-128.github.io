项目简介

这是一个可交互的神经网络分类器，专门帮助初学者直观理解"神经网络如何学会给数据分类"。你可以把它当成一个实验台：

  • 左侧画布：代表一个二维平面，横轴和纵轴是两个输入特征（例如 x 和 y）。点击即可放置不同颜色的数据点（红、黄、蓝、绿），每个点代表一个样本及其类别。
  • 右侧网络图：展示当前神经网络的连接结构。输入层有 2 个节点（接收坐标），输出层有 4 个节点（分别对应四种颜色），中间是你可以自由调整的隐藏层。
  • 训练过程：网络会用 tanh 激活函数和 softmax 输出，反复计算预测与真实标签的差距（交叉熵损失），并通过梯度下降不断更新权重，让背景色块逐渐划分出各类别的"势力范围"。
  • 动手实验：改变隐藏层的节点数或层数，再重新训练，观察网络结构如何影响分类能力。

试试看：先画一些分堆的点，再点击"开始训练"，背景颜色会慢慢分开。你还可以随时停止、修改网络、再训练！

技术栈

  • 纯原生 JavaScript：无任何框架或外部库
  • Canvas API：数据点与决策边界的像素级渲染
  • SVG：神经网络结构图的可缩放绘制
  • CSS 变量系统：统一的现代化配色方案

功能概览

模块 功能
目标图（Canvas） 点击添加四色数据点，训练时背景实时显示分类决策边界
神经网络图（SVG） 可视化网络层级结构，连线颜色/粗细编码权重值
训练控制 开始/停止训练，实时显示 loss、epoch、训练状态
网络修改 动态增删隐藏层节点/整层，点击图中节点快速选中

AI 辅助使用说明

本项目的开发过程中，AI 工具在以下环节提供了辅助：

1. 界面样式设计（CSS）

通过 GitHub Copilot 辅助生成现代化的卡片式布局、配色方案（CSS 变量体系）以及响应式圆角/阴影细节。Copilot 根据类名语义自动补全了大量样式声明，显著提升了 UI 开发效率。

2. 代码注释与文档

部分函数的功能说明、参数注释由 AI 辅助生成，帮助理清代码逻辑。

3. 调试辅助

在实现反向传播时，AI 工具帮助解释了梯度计算中的维度对齐问题，以及 softmax 数值稳定性（减去最大值防止溢出）的原理。

4. 优化建议

代码中注释提到的"修改成使用 Adam 优化器"方向，即来自 AI 对训练收敛速度的分析建议，为后续改进提供了思路。

核心算法（前向传播、反向传播、梯度下降、tanh/softmax 的数学实现）均通过自主查阅教材与资料后手写完成，AI 仅用于辅助理解与效率提升，未直接生成核心逻辑代码。

创意点

1. 纯前端神经网络

不依赖任何后端服务器或机器学习框架（如 TensorFlow/PyTorch），完全使用原生 JavaScript 从零实现多层感知机，包括前向传播、反向传播与梯度下降，打开浏览器即可运行。

2. 决策边界实时可视化

训练过程中以像素级粒度在画布背景上绘制每个类别的"势力范围"——通过遍历每个 14×14 像素格并进行预测着色，让抽象的数学分类边界变得直观可见。

3. 权重颜色-粗细双编码

网络连线同时使用颜色（蓝色\textcolor{blue}{蓝色}蓝色 = 正权重，红色\textcolor{red}{红色}红色 = 负权重）和线宽（粗细反映权重绝对值大小）来编码连接强度，使网络"知识"一目了然。

4. 可交互的网络结构编辑器

用户可在训练前后自由增删隐藏层节点或整层，点击网络图中的节点/层标签即可选中并修改，即时观察结构变化对分类效果的影响——将神经网络从"黑盒"变为可动手实验的"教具"。

5. 单文件零依赖架构

HTML、CSS、JavaScript 全部整合在一个文件中，无需 npm、构建工具或任何外部库，拷贝到任何设备即可使用，极大降低了初学者的使用门槛。

6. 异步训练 + UI 实时刷新

使用 requestAnimationFrame 配合 async/await 实现训练循环与界面渲染的协作调度，每 50 轮更新一次画面，既保证了训练速度又维持了视觉流畅度。

开发过程中遇到的困难

1. 反向传播的链式求导实现

最核心的困难在于手动推导多层网络的反向传播公式。需要逐层计算 δ\deltaδ（误差信号）并正确传递到前一层的每个神经元，尤其是 tanh⁡\tanhtanh 激活函数的导数 1−a21 - a^21−a2 要在正确的时机（激活值之后）使用。调试时经常出现梯度维度不匹配的问题。

2. 训练循环与 UI 响应的平衡

原始方案中训练循环会阻塞浏览器主线程，导致页面卡死无法点击"停止"按钮。通过将训练拆分为 async\mathrm{async}async 函数，每 NNN 轮用 await requestAnimationFrame() 让出控制权给浏览器，才实现了训练中的可中断交互。

3. 画布坐标归一化

数据点在画布上的像素坐标需要映射到神经网络输入的 [−1,1][-1, 1][−1,1] 区间。最初直接使用原始像素值导致 tanh⁡\tanhtanh 饱和区问题严重，后改为归一化到 [−1,1][-1, 1][−1,1] 后才使训练正常收敛。

4. Softmax 数值稳定性

直接对输出值取 Math.exp() 在数值较大时会溢出为 Infinity。查阅资料后采用"减去最大值"的稳定技巧 Math.exp(value - max)，既保持了相对大小关系又避免了溢出。

5. 网络结构动态修改后的状态同步

当用户在训练前后增删隐藏层节点或整层时，需要同步更新模型数据结构（layerSizes）、重新初始化权重、刷新 SVG 网络图、重绘 Canvas 背景，且不能丢失已有的数据点。这个过程涉及多个状态变量的协调，容易产生不一致。

6. 权重可视化的视觉校准

将权重数值映射到颜色透明度与线宽时，需要反复调整参数（透明度范围 0.18−0.850.18-0.850.18−0.85，线宽 1.2−6.21.2-6.21.2−6.2），使正负权重的视觉差异足够明显又不会过于刺眼。

使用方法

  1. 直接用浏览器打开 index.html
  2. 在左侧画布点击放置不同颜色的数据点
  3. 点击 “开始训练” 观察背景决策边界的变化
  4. 在右侧通过按钮或点击网络图修改隐藏层结构
  5. 点击 “刷新” 重新初始化网络，或继续训练观察结构变化的影响
